# Hvordan sette opp Git og GitHub

## Hva er Git?

[Git](https://git-scm.com/) er et verktøy for versjonskontroll som utviklere benytter. Med Git kan flere personer jobbe parallelt på samme kodebase. Verktøyet tillater også opprettelse av "branches", noe som muliggjør uavhengig arbeid. Når man er ferdig med endringene, kan man "merge" endringene med hovedversjonen av koden.

## Hva er GitHub?

[GitHub](https://github.com/) er en nettside som lar oss laste opp, dele og samarbeide på kodeprosjekter. I praksis fungerer GitHub som en skytjeneste som integreres med Git-programvaren.

I Webkom benytter vi Git for å hente koden vår fra GitHub til lokale datamaskiner. Her kan vi opprette en ny branch, gjøre ønskede endringer, og deretter laste opp den modifiserte branchen tilbake til GitHub. Dette gir andre muligheten til å gjennomføre "code review" før endringene eventuelt "merges" (legges til) med resten av koden.

## Laste ned Git

### Mac

<Callout type="info">
  Det er en mulighet for at `git` allerde er lastet ned på din Mac! Dette kan du sjekke ved å kjøre
  `git --version` i terminalen din.
</Callout>

Om du ikke har `git` anbefaler vi å laste det ned gjennom `brew`. Les [mer om brew her](https://brew.sh/).

<Steps>

### Last ned `brew`

Kopier dette inn i terminalen din, og trykk enter.

```sh
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### Last ned `git`

```sh
brew install git
```

### Sjekk at `git` er lastet ned

```sh
git --version
```

Da skal det printes ut noe som dette i terminalen din

```
$ git --version
git version 2.41.0
```

## Sette opp git

Etter du har lastet ned og innstallert `git` på datamaskinen din må du legge til navn og e-post.

<Callout>E-posten du legger til på Git burde være den samme som du bruker på GitHub.</Callout>

```sh
# For å legge til navn
git config --global user.name "<ditt navn>"

# For å legge til e-post
git config --global user.email "<din e-post>"
```

## Sette opp SSH

For å kunne laste opp endringer til GitHub må du sette opp SSH. SSH er en måte å autentisere seg på, og er en forkortelse for Secure Shell.

```sh
ssh-keygen -t ed25519 -C "<din e-post>"
```

Dette vil lage en SSH-nøkkel som du kan legge til på GitHub. Når du kjører kommandoen vil du bli spurt om hvor du vil lagre nøkkelen. Det er lurt å lagre den i `~/.ssh/id_ed25519` (trykk enter for å lagre den der).

Du kan lese din offentlige nøkkel ved å kjøre

```sh
cat ~/.ssh/id_ed25519.pub
```

Denne nøkkelen kan du legge til på GitHub. Gå til [GitHub](https://github.com) -> Trykk på profilbildet ditt -> Settings -> SSH and GPG keys -> New SSH key. Gi nøkkelen et navn (f.eks Min mac), og lim inn den offentlige nøkkelen din.

## Laste ned et repo

For å laste ned et repo fra GitHub kan du kjøre:

```sh
git clone git@github.com:<brukernavn>/<repo-navn>.git
```

For eksempel, om du ønsker å laste ned repoet til echo Webkom kan du kjøre:

```sh
git clone git@github.com:echo-webkom/echo-web
```

Dette vil laste ned repoet til din datamaskin og legge det i en mappe med samme navn som repoet.
